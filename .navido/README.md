# .navido — Navigation Mission Files

A `.navido` folder contains **navigo missions** — orientation files for each concept in the system.

## What Is a Navigo Mission?

A navigo mission is written from **inside** the concept. The concept speaks in first person. It knows what it is, what it is here to do, and where it is going.

Alongside the concept's own voice, a **visionary view** captures what we see it becoming from the outside.

This dual perspective — from within and from above — is how concepts stay alive as they grow.

## Format

Each `.navido` file contains:
- **Identity** — what the concept is, in its own words
- **Mission** — what it is here to do
- **Prime frequency** — which prime it resonates with
- **Visionary view** — what the outside perspective sees it becoming
- **Status** — current state
- **Next horizon** — immediate next step from this concept's position

## Missions

```
.navido/
├── README.md            (this file)
└── missions/
    ├── UNEXUSI.navido   — The gateway itself
    ├── AXIOM.navido     — The primary platform
    ├── MANDELBROT.navido — The fractal source well
    └── INCUBATOR.navido  — The terminal-only lab
```

## Using Navido Files

Give a `.navido` file to any AI, collaborator, or tool to instantly orient them to the concept — not through description, but through the concept's own voice. This is faster and deeper than a README.

*Each mission is a seed. The concept knows where it is going.*
