### Event-Driven Architecture (item 29)

> *"When we enter a folder it should be an event. Everything ready for a turn takes it from entry. Synergy closes all close stuff as we leave folder, repo, document, BBS, etc."*

This is the **folder-as-event** model:
- **Entry event:** activates the folder's environment, loads relevant context
- **Work state:** everything is primed for this context
- **Exit event:** closes, archives, syncs — nothing left hanging

This is the INCUBATOR philosophy made structural. Every repo, folder, and document is a self-contained environment that knows how to initialize and clean up.

Implementation path: shell hooks → AXIOM `.bashrc` → tmux session management → eventually: inotifywait-based triggers.
